# Terraform Labs for Cloud Infrastructure and AWS Security

This repository is a personal learning space for Infrastructure as Code (IaC) labs using Terraform, focused on cloud infrastructure practices and AWS security.

Author: Moises Miguel

## Purpose

This repo is intended to store and organize hands-on Terraform labs covering:

- Cloud infrastructure provisioning with Terraform
- AWS resource management
- Security best practices in AWS
- Terraform state management and drift detection
- Infrastructure automation and repeatability

## Repository Structure

- 1.Managing-Terrfaorm-State-and-drift/ - Labs related to Terraform state handling and drift detection

## What You Will Find Here

Each lab is designed to help practice:

- Writing Terraform configurations
- Managing providers and resources
- Understanding Terraform state
- Applying security principles to cloud infrastructure
- Improving reproducibility and maintainability of infrastructure code

## Prerequisites

Before using the labs, make sure you have:

- Terraform installed
- An AWS account
- AWS CLI configured
- Basic knowledge of cloud infrastructure concepts

## Suggested Workflow

1. Review the lab folder and its Terraform files.
2. Initialize Terraform with `terraform init`.
3. Review the planned changes with `terraform plan`.
4. Apply the configuration with `terraform apply`.
5. Destroy resources when no longer needed with `terraform destroy`.

## Notes

These labs are intended for learning and experimentation. Always follow AWS security best practices and avoid using production credentials in test environments.
